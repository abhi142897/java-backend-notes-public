# Reflection  
---

## What is reflection?  
- Reflection in Java is a feature that allows a program to inspect and manipulate classes, methods, fields, and constructors at runtime, even if they are private.
- It is part of the `java.lang.reflect package`.
- reflection is heavily used behind the scenes for:
  - Reading annotations like `@RestController`, `@Autowired`
  - Creating beans dynamically
  - Invoking methods dynamically
- Drawbacks of Reflection
  - Slower than direct method calls
  - Breaks encapsulation (can access private members)
  - Harder to debug
  - Security restrictions in some environments

## How Reflection Works (Basic Flow)
1. Get the Class object  
2. Get metadata (fields, methods, constructors)  
3. Access/modify/invoke them dynamically  

## Important Reflection Classes
| Class         | Purpose                     |
| ------------- | --------------------------- |
| `Class`       | Represents class metadata   |
| `Field`       | Represents class fields     |
| `Method`      | Represents methods          |
| `Constructor` | Represents constructors     |
| `Modifier`    | Check public/private/static |

## How to do reflection of classes?
To reflect the class, we first need to get an object of `Class`.  

### What is class Class?  
- Instance of the class `Class` represents classes during runtime.
- JVM creates one `Class` object for each and every class which is loaded during runtime.
- This Class object, has meta data information about the particular class like its method, fields, constructor etc.

### How to get the Class object?
There are 3 ways:  
Assume that we have an Employee class -
```java
public class Employee {

    private String name;
    public int age;

    public Employee() {
    }

    public Employee(String name) {
        this.name = name;
    }

    public void sayHello() {
        System.out.println("Hello " + name);
    }

    private void privateMethod() {
        System.out.println("Private Method Called");
    }
}
```

1. Using `forName()` method  
```java
public static void main(String[] args) throws ClassNotFoundException {
    Class employee = Class.forName("Employee"); 
}
```

2. Using `.class`  
```java
public static void main(String[] args) throws ClassNotFoundException {
    Class employee = Employee.class;
}
```

3. Using `getClass()` method
```java
public static void main(String[] args) throws ClassNotFoundException {
    Employee employee = new Employee();
    Class employeeClazz = employee.getClass();
}
```

### Now let's see how to do reflection of classes
```java
    public static void main(String[] args) throws ClassNotFoundException {
        Class employeeClass = Employee.class;
        System.out.println(employeeClass.getName()); // test.relfectionPractice.Employee
        System.out.println(employeeClass.getSimpleName()); // Employee
        System.out.println(employeeClass.getPackage().getName()); // test.reflectionPractice
        System.out.println(employeeClass.getConstructors().length); // 2
        System.out.println(Modifier.toString(employeeClass.getModifiers())); // public
    }
```

## Methods available in Class object
1. Getting Class Information  

| Method             | Purpose                                    |
| ------------------ | ------------------------------------------ |
| `getName()`        | Returns fully qualified class name         |
| `getSimpleName()`  | Returns class name without package         |
| `getPackageName()` | Returns package name                       |
| `getSuperclass()`  | Returns parent class                       |
| `getInterfaces()`  | Returns implemented interfaces             |
| `getModifiers()`   | Returns modifiers (public, abstract, etc.) |
| `isInterface()`    | Checks if it’s an interface                |
| `isEnum()`         | Checks if enum                             |
| `isAnnotation()`   | Checks if annotation                       |
| `isArray()`        | Checks if array                            |
| `isPrimitive()`    | Checks if primitive type                   |

2. Getting Constructors 

| Method                                           | Purpose                     |
| ------------------------------------------------ | --------------------------- |
| `getConstructors()`                              | Public constructors         |
| `getDeclaredConstructors()`                      | All constructors            |
| `getConstructor(Class<?>... paramTypes)`         | Specific public constructor |
| `getDeclaredConstructor(Class<?>... paramTypes)` | Any specific constructor    |


3. Getting Methods  

| Method                                                   | Purpose                                  |
| -------------------------------------------------------- | ---------------------------------------- |
| `getMethods()`                                           | All public methods (including inherited) |
| `getDeclaredMethods()`                                   | All methods of the class                 |
| `getMethod(String name, Class<?>... paramTypes)`         | Specific public method                   |
| `getDeclaredMethod(String name, Class<?>... paramTypes)` | Any specific method                      |


4. Getting Fields  

| Method                          | Purpose               |
| ------------------------------- | --------------------- |
| `getFields()`                   | Public fields         |
| `getDeclaredFields()`           | All fields            |
| `getField(String name)`         | Specific public field |
| `getDeclaredField(String name)` | Any specific field    |


5. Creating Objects  

| Method                                   | Purpose                                 |
| ---------------------------------------- | --------------------------------------- |
| `newInstance()` ⚠️ (Deprecated)          | Create object using default constructor |
| `getDeclaredConstructor().newInstance()` | Recommended way                         |


6. Annotation Related  

| Method                                             | Purpose                       |
| -------------------------------------------------- | ----------------------------- |
| `getAnnotations()`                                 | All annotations               |
| `getDeclaredAnnotations()`                         | Annotations declared on class |
| `isAnnotationPresent(Class<? extends Annotation>)` | Check if annotation exists    |
| `getAnnotation(Class<T>)`                          | Get specific annotation       |


7. Class Loading Related 

| Method                      | Purpose                |
| --------------------------- | ---------------------- |
| `forName(String className)` | Load class dynamically |
| `getClassLoader()`          | Get class loader       |


8. Utility Methods  

| Method                           | Purpose                    |
| -------------------------------- | -------------------------- |
| `cast(Object obj)`               | Cast object safely         |
| `asSubclass(Class<U> clazz)`     | Narrow type                |
| `isAssignableFrom(Class<?> cls)` | Check inheritance relation |


### Difference Between `getMethods()` vs `getDeclaredMethods()`  
| Feature                       | `getMethods()` | `getDeclaredMethods()`   |
| ----------------------------- | -------------- | ------------------------ |
| Access level                  | Only public    | All access modifiers     |
| Includes inherited methods    | ✅ Yes          | ❌ No                     |
| Includes private methods      | ❌ No           | ✅ Yes                    |
| Includes Object class methods | ✅ Yes          | ❌ No                     |
| Used mostly in                | API inspection | Framework/internal logic |

**Note: Same Concept Applies to `getFields()` vs `getDeclaredFields()`**

## Mini Practical Example Using Major `Class` Methods
- `Employee` class
```java
@MyAnnotation("ServiceClass")
public class Employee {

    private String name;
    public int age;

    public Employee() {
    }

    public Employee(String name) {
        this.name = name;
    }

    public void sayHello() {
        System.out.println("Hello " + name);
    }

    private void privateMethod() {
        System.out.println("Private Method Called");
    }
}


@Retention(RetentionPolicy.RUNTIME)
@interface MyAnnotation {
    String value();
}
```
- `ReflectionDemo` Class for `Employee` class 
```java
public class ReflectionDemo {

    public static void main(String[] args) throws Exception {

        // 1️⃣ Load Class dynamically
        Class<?> clazz = Class.forName("test.reflectionPractice.Employee");

        // 2️⃣ Basic Class Info
        System.out.println("Class Name: " + clazz.getName());
        System.out.println("Simple Name: " + clazz.getSimpleName());
        System.out.println("Package: " + clazz.getPackageName());
        System.out.println("Superclass: " + clazz.getSuperclass());
        System.out.println("Is Interface? " + clazz.isInterface());

        // 3️⃣ Check Annotation
        if (clazz.isAnnotationPresent(MyAnnotation.class)) {
            MyAnnotation annotation = clazz.getAnnotation(MyAnnotation.class);
            System.out.println("Annotation Value: " + annotation.value());
        }

        // 4️⃣ Get Constructors
        Constructor<?>[] constructors = clazz.getDeclaredConstructors();
        for (Constructor<?> c : constructors) {
            System.out.println("Constructor: " + c.getName());
        }

        // 5️⃣ Create Object
        Object obj = clazz.getDeclaredConstructor(String.class)
                .newInstance("Abhishek");

        // 6️⃣ Get Fields
        Field[] fields = clazz.getDeclaredFields();
        for (Field f : fields) {
            System.out.println("Field: " + f.getName());
        }

        // Access private field
        Field nameField = clazz.getDeclaredField("name");
        nameField.setAccessible(true);
        nameField.set(obj, "John");

        // 7️⃣ Get Methods
        Method[] methods = clazz.getDeclaredMethods();
        for (Method m : methods) {
            System.out.println("Method: " + m.getName());
        }

        // Invoke public method
        Method method = clazz.getDeclaredMethod("sayHello");
        method.invoke(obj);

        // Invoke private method
        Method privateMethod = clazz.getDeclaredMethod("privateMethod");
        privateMethod.setAccessible(true);
        privateMethod.invoke(obj);

        // 8️⃣ Inheritance Check
        System.out.println("Is Object assignable? " +
                Object.class.isAssignableFrom(clazz));
    }
}
```
- Output
```txt
Class Name: test.reflectionPractice.Employee
Simple Name: Employee
Package: test.reflectionPractice
Superclass: class java.lang.Object
Is Interface? false
Constructor: test.reflectionPractice.Employee
Constructor: test.reflectionPractice.Employee
Field: name
Field: age
Method: sayHello
Method: privateMethod
Hello John
Private Method Called
Is Object assignable? true
```

## Why Reflection is Slower?  
- No Compile-Time Optimization
- No Inlining (Major Reason). Always goes through `Method.invoke()`
- Extra Indirection Layer
```text
Method.invoke() → Access checks → Argument handling → Actual method call
```
- Dynamic Type Resolution
- Reflection is typically 2x to 10x slower

## ✅ Why Frameworks Still Use It
Frameworks like Spring use reflection for:  
- Dependency injection
- Annotation processing
- Object creation

👉 But:  
- Done mostly at startup time
- Cached aggressively



## Breaking Singleton Using Reflection
### Typical Singleton Implementation
```java
class Singleton {

    private static final Singleton instance = new Singleton();

    private Singleton() { }   // private constructor

    public static Singleton getInstance() {
        return instance;
    }
}
```

### How Reflection Breaks It
Reflection can:  
 - Access private constructor
 - Force it accessible
 - Create another object
 ```java
 import java.lang.reflect.Constructor;

public class BreakSingleton {
    public static void main(String[] args) throws Exception {

        Singleton instance1 = Singleton.getInstance();

        Constructor<Singleton> constructor =
                Singleton.class.getDeclaredConstructor();

        constructor.setAccessible(true);   // break private access

        Singleton instance2 = constructor.newInstance();

        System.out.println(instance1 == instance2);  // false
    }
}
```

### How to Prevent Reflection Breaking Singleton
1. Throw Exception in Constructor
```java
class Singleton {

    private static final Singleton instance = new Singleton();

    private Singleton() {
        if (instance != null) {
            throw new RuntimeException("Use getInstance()");
        }
    }

    public static Singleton getInstance() {
        return instance;
    }
}
```

2. Best Solution → Use ENUM Singleton  
```java
enum Singleton {
    INSTANCE;
}
```
🚨 Why ENUM is Safe?
- JVM ensures only one instance
- Reflection cannot create enum instances
- Serialization safe
- Thread safe
- Even reflection will throw:
```sh
java.lang.IllegalArgumentException: Cannot reflectively create enum objects
```
